테이블 착석 방식 비교 A / B / C

각 방식의 시각적 결과 비교 — 2026-04-11

A

컴포짓 스프라이트

테이블+손님을 아이소메트릭 뷰로 하나의 PNG에 합성

  • 뷰 완전 일치
  • 구현 단순
  • 품질 높음
  • 손님종 × 테이블레벨 조합 필요
  • 리액션 애니 어려움
B

High Top-Down 캐릭터

캐릭터를 테이블과 같은 각도로 생성 (정수리 시점)

  • 뷰 각도 일치
  • depth split 필요 없음
  • 캐릭터 표정/감정 표현 어려움
  • 이동 애니가 어색
C

정면 캐릭터 + 테이블 앞에 배치

캐릭터를 테이블 앞(south)에 놓고 상체가 테이블 위로 겹치게

  • 캐릭터 표정 선명
  • 애니 자유도 높음
  • 구현 빠름
  • 뷰 미스매치
  • 스타듀밸리 스타일 (비현실적)
